# 自动化外贸开发系统

这是一个本地可运行的外贸自动化开发管理系统。核心流程是：输入产品和目标市场，系统生成搜索任务；把 Google、LinkedIn、目录、海关数据或第三方采集结果粘贴/导入后，系统解析成线索池，再生成邮箱/WhatsApp 触达内容、发信队列、跟进任务，并通过「管理」模块统一管理活动、任务、审批、账号、规则和线索阶段。

## 已具备

- AI 自动获客 Agent：用一句话下达开发任务（如"帮我找美国和加拿大做户外家具批发的进口商，规模 20 人以上，自动发开发信和 WhatsApp"），系统解析为任务卡片（Claude 或本地规则），确认后自动执行 寻客漏斗（抓取→画像匹配→验证→去重→评分入围）→ 触达审批（逐条 / 批量抽审 / 全自动三种模式）→ 自动开发（AI 初轮应答标准问题、敏感话题转人工、opt-out 即时黑名单、全程留痕）→ 意向分级移交（🔥热=询价要样及转人工立即接管、🌤温=培育、❄️冷=换角度或黑名单），并按市场回复率给出"效果回流"倾斜建议。
- 开发活动配置：产品、目标市场、客户类型、卖点、认证、署名和每日开发上限。
- 搜索自动化：按进口商/经销商、批发目录、采购负责人、LinkedIn、零售商供应商入口、协会名录、海关数据、竞品渠道生成搜索任务。
- 搜索结果导入：粘贴公司官网、Google 结果、LinkedIn 公司页、邮箱、电话或 CSV 行，自动解析为待审核线索。
- 潜客队列：管理公司、市场、网站、来源、角色、评分、采购信号、审核状态。
- AI 客户评分：按官网真实性、联系方式、采购信号、客户类型和互动信号（回复/打开/触达）加权计算成交概率，输出 A/B/C/D 优先级与可解释的因子明细。
- 质量分排序与一键入队：潜客队列可按质量分/最新/市场排序、按 A/B/C/D 质量分筛选，顶部有各级数量概览；「一键入队优质客户」把所有 A/B 级、有联系方式、未入队/未回复/不在冷却的优质客户一次性加入触达队列（按钮实时显示可入队数量，首封仍待你审批发送）。
- 智能找联系方式：本地按域名+决策人生成多个候选邮箱（带置信度与模式，可一键设为主邮箱）；「AI 找联系人」按真实源优先（Hunter/Apollo/Dropcontact via 邮箱查找 Webhook，返回验证过的联系方式）→ Claude 推测决策人/候选邮箱/公司画像/匹配度 → 本地规则兜底，来源有徽章标注（真实验证/AI 推测/规则推测）。
- Claude 联网找客户：搜索页「Claude 联网找客户」按钮，用 Claude 内置联网搜索（web_search）按当前活动的市场/客户类型/产品/关键词，直接搜出真实存在的目标采购商并核验官网，解析进线索池走漏斗（需在设置配置 Claude API Key）。自动排除 alibaba/amazon 等平台与目录站本身。
- 粘贴/导入解析增强：一次粘贴尽量多抽公司——支持一行含多个 URL 自动拆分、CSV 表头自动识别跳过、全文域名兜底扫描（正文里任何真实域名都能补成线索），并自动跳过 google/linkedin/alibaba 等平台域名；散文行内夹域名时用域名反推干净公司名。
- 找相似客户：潜客详情「找相似客户」按钮，以选中这家为样本，配 Claude 时联网找同市场/同品类/相近规模的真实相似公司，未配 Claude 时用本地规则按其特征生成一批（清晰去重，展示名不含内部标记）。
- 官网一键深挖联系人：潜客详情「官网深挖联系人」按钮，Claude 联网翻这家官网的 About/Team/Contact/Wholesale 页与领英，找真实采购决策人的姓名/职位/邮箱/电话（找到的标 verified、找不到按域名模式给候选），来源徽章「联网核实」（需配置 AI 引擎）。
- 一键批量补全联系方式：潜客队列「批量补全联系方式」按钮，对所有缺邮箱/待补全/新发现的线索依次跑「AI 找联系人」链路（真实源 Webhook→Claude→本地规则），带进度日志，省去逐条点击。
- 竞品渠道反查：搜索页「竞品渠道反查」，粘贴竞品的 Where-to-buy / 经销商列表 / dealer locator 页面链接，Claude 联网抽出他家所有经销商/分销商作为你的高意向线索进池（需配置 AI 引擎）。
- 每日自动联网找新客户：Agent 任务的周期补量可勾选「联网找真实客户」，配合自动驾驶到周期用 Claude 联网搜真实客户补量（未配 Claude 或联网无果时自动降级到采集 Webhook/生成器）。
- Webhook 联调：每个渠道 Webhook 可一键测试连接（含状态码与延时），Webhook 模式下把邮件/WhatsApp/CRM 队列真实派发给外部服务，并记录活动日志。
- 自动驾驶：一键开启全流程自动流转——新线索自动补全验证、高分自动入队（遵守评分阈值/日上限/冷却期）、到期邮件批量发送并回传、跨渠道接力自动执行、客户回复自动生成 AI 草稿送审批。
- 流程规则：客户回复即自动取消剩余触达序列（回复即停）；询价/要样/砍价等意图自动推进商机到「询盘」；冷却期内不重复触达同一客户；审批通过即自动发送/派发。
- 界面体验：导航按「获客→触达→转化」流程分组并带待办徽标（未读回复/待审批/待发送）；所有操作即时 Toast 反馈；控制台新手引导清单五步上手；空页面提供下一步跳转按钮。
- 效率工具：Ctrl+K 全局命令面板（搜客户/页面/动作直达）；深色模式一键切换并记忆；收件箱内直接手写快捷回复（邮件/WhatsApp 双渠道）；CRM 卡片点击弹出详情抽屉（改阶段/金额、看动态、跳会话）；数据分析支持 全部/近7天/近30天 时间范围。
- 资料补全：模拟联系人、邮箱规则、公司规模和置信度。
- 邮箱验证：基础格式验证和可发信状态流转。
- 开发信序列：首封、价值跟进、样品/案例、最后触达四封英文邮件。
- WhatsApp 开发：生成号码、WhatsApp 话术、wa.me 打开聊天链接和待确认队列。
- 发信队列：把高分潜客或单个潜客加入发送队列。
- 全渠道统一收件箱：把邮件与 WhatsApp 消息按客户汇聚成同一条时间线，标记客户回复、未读和渠道来源。
- 跨渠道接力：邮件未回 N 天自动转 WhatsApp 触达，WhatsApp 未覆盖回退邮件；客户一旦回复，该会话自动停止接力。
- AI 助手：客户回复后自动识别意图（询价/要样/砍价/交期/认证/拒绝等）、生成个性化回复建议、给出对话摘要与下一步动作，可一键复制或采用为回复。
- 回信风险识别：自动扫描客户来信中的风险事项（付款风险/疑似诈骗/制裁合规/知识产权/利润风险/样品滥用/信息不一致），分高中低三级、给出证据与应对建议，展示在收件箱 AI 面板与 Agent 热意向卡；识别到高风险时初轮应答护栏自动转人工。本地规则 + Claude 双引擎。
- 周期任务：Agent 任务可设为每天/每周/每月自动补充 N 个新线索，配合自动驾驶到周期自动执行一遍寻客漏斗，也可「立即补充一批」；真实部署走搜索采集 Webhook + 外部 cron。
- AI 引擎（Claude API）：在「设置 → AI 引擎」填入 Anthropic API Key 后，意图识别、回复建议、对话摘要升级为真实大模型语义分析，并解锁「Claude 深度写信」（按客户画像生成整套个性化开发信序列）；未配置或调用失败时自动降级为本地规则。
- CRM 商机看板：线索 → 已触达 → 已回复 → 询盘 → 报价 → 成交 拖拽式管道，超期跟进标红、管道金额和成交率实时统计。
- 数据分析看板：渠道漏斗、邮件/WhatsApp 渠道对比、双渠道接力效果、市场表现、话术回复率排行，北极星指标为有效询盘数/月。
- 跟进任务：按 3、7、14 天生成自动跟进任务。
- 管理后台：活动管理、自动化任务中心、审批中心、渠道账号、规则配置、线索阶段漏斗。
- 导出：搜索式、潜客、发信队列、WhatsApp 队列和完整 JSON 数据。
- Webhook 接口位：预留搜索采集、邮箱补全/验证、发信、WhatsApp Business 和 CRM 同步接口。

## AI 自动获客 Agent

「AI Agent」是系统的总入口——五步闭环：任务解析 → 自动寻客 → 触达审批 → 自动开发 → 意向移交。

1. **下达任务**：用一句话描述目标客户。配置了 Claude API（设置 → AI 引擎）时由大模型解析并自动扩展英文行业关键词与当地术语；未配置时用本地规则解析（国家、客户类型、日上限、渠道均可识别）。
2. **任务卡片确认**：解析结果以可编辑卡片回显（产品/市场/类型/关键词/上限/渠道），确认后才执行，避免理解偏差浪费额度。同时选择审批模式——逐条审批（默认，冷启动推荐）/ 批量抽审 / 全自动（自动开启自动驾驶）。
3. **自动寻客**：确认后生成搜索任务；导入线索（粘贴 / Webhook / 演示数据）即触发过滤漏斗：原始抓取 → 画像匹配（市场过滤）→ 验证 → 去重 → AI 评分入围（受每日上限约束）。
4. **触达审批**：每个入围客户生成审批卡——是谁、为什么值得开发（评分因子）、邮件首触与 WhatsApp 话术预览；逐条通过 / 跳过，或一键全部通过。抽审与全自动模式自动通过入队。
5. **自动开发（AI 初轮应答，带护栏）**：在「AI Agent → 第 4 步」开启后，客户回复标准问题（发目录/要样品/问 MOQ/交期/认证/一般价格）时 AI 自动答复，并明确告知详细报价由销售同事跟进；可填写「产品知识库/FAQ」喂给 AI。三重护栏：**涉及具体报价折扣、账期、独家代理、合同条款等敏感话题一律不擅自答复，立即转人工**；客户 opt-out（unsubscribe/退订等）即时加入黑名单并取消全部待发触达、之后不再入队；所有 AI 应答与转人工动作留痕在面板可查。配置 Claude API 时回复内容由大模型基于知识库生成，否则用护栏模板。
6. **意向移交**：客户回复后按意图自动分级——询价/要样/MOQ/交期/认证及被转人工的敏感话题为 🔥 热意向（卡片附对话摘要，点「接管会话」直达收件箱）；回复未询价为 🌤 温（培育）；已读不回、明确拒绝与 opt-out 为 ❄️ 冷。有足够数据时给出"效果回流"建议（哪个市场回复率显著更高，建议倾斜）。

合规说明：本系统不做任何协议破解或绕过反爬的抓取；线索来源为公开信息的人工/Webhook 导入，「演示数据」仅用于体验流程。

## 使用

直接用浏览器打开 `index.html`。

推荐流程：

1. 在「控制台」填写产品、市场、客户类型和卖点。
2. 点击「生成开发计划」得到搜索任务。
3. 打开搜索任务，复制真实公司官网、LinkedIn 公司页、邮箱、电话或 CSV 行。
4. 回到「搜索」里的「搜索结果导入」，粘贴结果并点击「解析为线索」。
5. 在「潜客」里审核线索、补全资料、验证邮箱和生成 WhatsApp。
6. 在「邮件」「WhatsApp」「队列」里生成触达内容、加入队列和导出。
7. 在「管理」里保存活动、运行任务、审批 WhatsApp 队列、配置自动化规则和导出管理数据。

## 管理后台

「管理」模块用于把自动化流程变成可运营的管理系统：

- 活动管理：保存当前开发活动，切换不同产品/市场活动。
- 自动化任务中心：管理搜索采集、资料补全、邮箱/号码验证、话术生成、入队限流、CRM 同步等任务。
- 审批中心：集中处理 WhatsApp 待确认消息、话术审核和低分线索风控。
- 渠道账号：查看 Email、WhatsApp、Search API、CRM 的使用量和接入状态。
- 自动化规则：配置邮件日上限、WhatsApp 日上限、最低线索评分、冷却天数、WhatsApp 是否必须审批。
- 线索阶段漏斗：查看新发现、已丰富、邮箱有效、已入队、已回复等阶段分布。

## 自动驾驶（全流程自动化）

顶栏「自动驾驶」开关开启后，系统按固定节拍（演示模式每 8 秒一拍，对应真实系统的小时级调度）自动执行整条获客流水线，无需逐步点击：

1. 新导入线索自动补全资料 + 验证邮箱/号码
2. 达到评分阈值的线索自动加入发信队列（遵守每日上限与冷却期）
3. 到期邮件批量发送（Webhook 模式走真实发信服务），并自动生成送达/打开回传
4. 已审批的 WhatsApp 自动发送/派发
5. 跨渠道接力自动执行：邮件未回转 WhatsApp、WhatsApp 未覆盖回退邮件
6. 客户回复后自动生成 AI 回复草稿，进入审批中心等待人工放行

**发送始终需人工审批**：自动驾驶只负责 找客户→补全联系方式→验证→评分→把高分线索备到「待发送 / 待审批卡」，绝不自动发出。发送由人工触发——Agent 审批卡「通过并发送」、队列「发送今日到期」、或审批中心「全部通过」。Agent 审批模式为 逐条审批 / 批量审批（均需人工过一遍，已移除全自动发送模式）。

事件驱动的自动规则（无论自动驾驶是否开启都生效）：

- **回复即停**：客户一回复，其剩余未发的邮件序列和 WhatsApp 队列自动取消，不再打扰。
- **意图推进商机**：AI 识别到询价/要样/砍价/交期/MOQ/认证等意图时，商机自动推进到「询盘」（北极星指标自动增长）；识别到拒绝则转入培育名单。
- **冷却期**：同一客户在「自动化规则」设定的冷却天数内不会被重复加入触达队列——这也让自动驾驶默认「邮件先行，WhatsApp 只走接力」，符合合规理念。
- **导入即处理**：自动驾驶开启时，粘贴导入搜索结果后立即自动补全、验证、入队。

「管理」页的自动化任务中心也已接入真实执行：「运行待执行」会依次真实运行采集（Webhook 模式）、补全、验证、话术、入队、CRM 同步，并显示每项的实际处理结果。

## AI 客户评分

「潜客」列表和详情面板会展示每个客户的 AI 成交概率与优先级：

- 加权因子模型：基础分 + 官网真实性、邮箱质量、WhatsApp 号码、采购角色/信号、客户类型匹配、资料置信度，以及权重最高的互动信号（已回复 > 已打开 > 已触达）。
- 成交概率与分级：综合得分映射为 0–99% 成交概率和 A/B/C/D 优先级，支持「高分客户优先触达」。
- 可解释：详情面板逐项列出每个因子的加分与短板（如「缺邮箱」「尚未触达」），评分随触达和客户回复实时变化。

本地为规则加权实现；接入真实数据（海关进口记录、企业库、行为回传）后，同一套因子框架可替换为机器学习模型输出。

## 全渠道统一收件箱与跨渠道接力

「收件箱」模块把发信队列（邮件）、WhatsApp 队列和客户回复汇聚到同一条客户时间线，是本系统的核心差异点：

- 会话列表：按客户聚合，显示已用渠道、是否已回复、未读数、是否已接力。
- 客户时间线：邮件、WhatsApp、客户回复按时间合并，完整还原沟通历史。
- 跨渠道接力规则：
  - 「邮件未回自动转 WhatsApp」：对已发邮件但未回复、且有号码的客户，自动生成 WhatsApp 接力触达（默认仍走「待人工确认」，遵守审批规则）。
  - 「WhatsApp 未覆盖回退邮件」：对只在 WhatsApp 触达、但有邮箱的客户，自动补一封接力邮件。
  - 未回天数可配置；把天数设为 0 可立即演示接力。
  - 客户一旦回复，会话标记「已回复」，自动停止对该客户的接力。
- 演示：每个未回复会话可点「模拟客户回复」验证「回复即停」逻辑，也可对单个会话手动「转 WhatsApp 接力 / 转邮件接力」。

接入真实 WhatsApp Business / 邮件 API 后，接力生成的队列条目可直接通过「设置」里的 Webhook 交给模板消息服务或 SMTP 服务发送。

### AI 助手（意图识别 + 回复建议）

当会话收到客户回复时，收件箱时间线会出现「AI 助手」面板：

- 意图识别：按关键词优先级（拒绝 > 议价 > 样品 > 询价 > 交期 > MOQ > 认证）判定客户最新意图并给出置信度，时间线里的每条客户回复也会标注意图标签。
- 对话摘要：一句话概括触达次数、回复数、最新意图和当前商机阶段，并给出下一步动作建议。
- 回复建议：按意图和活动信息（产品、卖点、认证、署名）生成个性化英文回复，可「复制建议」或「采用为回复」——采用后按客户回复的渠道自动加入邮件或 WhatsApp 队列。

默认是规则启发式实现；在「设置 → AI 引擎」切换到 Claude API 并填入 Anthropic API Key（仅存本机浏览器 localStorage，不上传任何服务器）后：

- 客户回复会自动发给 Claude 做**语义级**意图判断（结构化输出保证结果可直接入库），面板会显示「Claude · 模型名」来源徽章，摘要与建议回复由真实大模型基于完整对话上下文生成。
- 邮件页新增「Claude 深度写信」：按客户公司、市场、角色、采购信号生成整套 4 封个性化开发信，替代模板填空。
- 模型可选 Claude Opus 4.8（默认，最强）/ Sonnet 5 / Haiku 4.5；「测试连接」一键验证 Key 与延时。
- 浏览器直连 Anthropic API（带官方 `anthropic-dangerous-direct-browser-access` 头）；任何调用失败都会记录日志并自动回退本地规则，不阻塞流程。

获取 API Key：https://platform.claude.com/ → API Keys。注意：Key 属于敏感凭证，本模式适合个人/本机使用；团队部署建议改为服务端代理转发。

## CRM 商机看板

「CRM」模块把潜客变成可管理的商机管道，是承接收件箱的「管客户」环节：

- 六阶段看板：线索 → 已触达 → 已回复 → 询盘 → 报价 → 成交，拖拽卡片即可推进商机。
- 自动推进：加入邮件/WhatsApp 队列自动进入「已触达」，客户回复自动进入「已回复」，无需手动同步。
- 超期提醒：卡片显示下次跟进日期，超期未跟进标红，触达后未安排跟进标黄。
- 管道统计：总商机、询盘及以上、报价中、成交率、在谈金额、超期跟进数实时汇总；每列显示商机数与预估金额。
- 导出：一键导出管道 CSV（公司、阶段、评分、金额、联系方式、下次跟进）。

预估金额目前按线索评分自动生成，接入真实 CRM（Twenty、HubSpot 等）后可通过 Webhook 双向同步真实金额与阶段。

## 数据分析看板

「分析」模块把触达数据变成可决策的运营指标：

- 北极星指标：有效询盘数/月（进入询盘及以上阶段的客户数），而非发送量。
- 渠道漏斗：触达 → 送达 → 打开 → 回复 → 询盘，按客户去重并显示各级环比转化率。
- 渠道对比：邮件与 WhatsApp 各自的触达/送达/打开(已读)/回复率并列，直观体现双渠道差异。
- 双渠道接力效果：对比单渠道与双渠道（接力覆盖）客户的回复率，量化「双渠道回复率高一个量级」的产品卖点。
- 市场表现：按市场统计触达、回复、询盘和回复率排名。
- 话术回复率排行：按邮件/WhatsApp 模板统计发送与回复率，沉淀公司级高效话术。

送达率与打开率在本地模拟中由「模拟送达/打开回传」按钮按线索质量生成；接入邮件服务商与 WhatsApp Business API 后，这些数据由真实的 delivery / open / read 回调经 Webhook 写入。

## WhatsApp 开发

系统会为潜客生成 WhatsApp 联系方式、短话术和打开聊天链接。建议用于已授权、展会名片、询盘、老客户或已经通过邮件/LinkedIn 建立上下文的客户。

默认 WhatsApp 队列状态是「待人工确认」，避免直接批量冷发导致账号风控。后续如果接 WhatsApp Business API，可以把队列通过 Webhook 交给模板消息服务。

## 接入真实自动化

当前版本默认是本地管理系统，不会绕过浏览器或平台规则去抓取公司，也不会真实发送邮件/WhatsApp。没有接 API 时，它负责生成搜索任务、解析导入结果、管理线索和队列。

要接真实流程，可以在「设置」里切到 Webhook 模式，并填入：

- 搜索采集 Webhook：接 Google Custom Search、SerpAPI、Bing Search API、Apify、n8n 或 Activepieces。
- 邮箱查找/验证 Webhook：接 Hunter、Apollo、Dropcontact、ZeroBounce 或 NeverBounce。
- 发信 Webhook / SMTP 服务：接自建发信服务、n8n SMTP、SendGrid、Mailgun 或企业邮箱 API。
- WhatsApp Business Webhook：接 WhatsApp Business Platform、360dialog、Twilio、WATI、n8n 或 Activepieces。
- CRM 同步 Webhook：接 Twenty、悟空 CRM、EspoCRM、HubSpot 或自定义数据库。

### 连接测试与派发

每个 Webhook 输入框下有「测试连接」按钮：点击会向该地址 POST 一个 `{"type":"<连接名>","ping":true,...}` 的探测请求，并在旁边显示状态（正常/失败/未配置 + HTTP 状态码 + 延时）。

「Webhook 派发」面板的「派发待处理队列」按钮：

- Webhook 模式下：把待发送邮件批量交给发信服务、已审批 WhatsApp 交给 Business API、全部潜客同步到 CRM，成功后才把队列条目标记为已发送；失败则保留原状态并记录错误。
- 本地模式下：仅做本地模拟发送。
- 所有调用（含搜索采集与连接测试）都记录在下方的「Webhook 活动日志」，可追溯地址、状态码和耗时。

派发时发往各 Webhook 的载荷：

- 发信：`{ "type":"send", "emails":[{company,email,subject,body,...}] }`
- WhatsApp：`{ "type":"whatsapp", "messages":[{company,phone,message,url,...}] }`
- CRM：`{ "type":"crm", "prospects":[{company,market,email,phone,stage,score}] }`

> 浏览器直连 Webhook 受同源策略限制，目标地址需允许跨域（CORS），或在自动化平台（n8n/Make/Activepieces）侧开启。

Webhook 返回潜客时建议包含：

```json
{
  "prospects": [
    {
      "company": "Example Imports",
      "market": "United States",
      "website": "example.com",
      "contactName": "Anna Smith",
      "role": "Sourcing Manager",
      "email": "sourcing@example.com",
      "phone": "+12345678900",
      "source": "Google",
      "score": 82
    }
  ]
}
```

### 真实数据源接口（搜索采集 Webhook）

在「设置 → 搜索采集 Webhook」填入自动化平台（n8n / Make / Activepieces）或自建服务的地址后，系统在寻客/周期任务时会向该地址 POST 一个搜索请求，期望同步返回潜客数组。

系统发出的请求载荷（`campaign` 为当前活动配置，`searchPlan` 为已生成的搜索式数组）：

```json
{
  "type": "search",
  "sentAt": "2026-07-05T00:00:00.000Z",
  "campaign": {
    "product": "stainless steel water bottle",
    "markets": "United States, Germany",
    "customerType": "进口批发商",
    "valueProps": "...",
    "dailyLimit": 20
  },
  "searchPlan": [
    { "channel": "Google", "market": "United States", "query": "\"stainless steel water bottle\" (\"importer\" OR \"distributor\") \"United States\" -alibaba" }
  ]
}
```

期望返回（与上文"Webhook 返回潜客"结构一致，`prospects` 数组即可）：

```json
{
  "prospects": [
    { "company": "Acme Drinkware", "market": "United States", "website": "acmedrinkware.com", "email": "buyer@acmedrinkware.com", "source": "SerpAPI", "score": 80 }
  ]
}
```

#### n8n 示例工作流

- **Google/SerpAPI 找官网**：Webhook 触发 → HTTP Request 调 SerpAPI（`q` 用 `campaign.product`+`campaign.markets` 拼搜索式，或直接复用传入的 `searchPlan[].query`）→ Function 节点解析 organic_results 的 title/link 成 `{company, website, source:"SerpAPI"}` → Respond to Webhook 返回 `{prospects:[...]}`。
- **海关数据核验进口商**：Webhook → HTTP Request 调海关数据服务（如 ImportGenius / 采购商数据 API）按品类查有真实进口记录的买家 → 映射成 prospects（`buyingSignal` 写"有 N 条进口记录"）→ 返回。
- **Apollo/Hunter 补联系人**：把上一步的 website 传给邮箱补全 Webhook（Apollo People Search / Hunter Domain Search）→ 返回 `contactName/role/email` 合并进 prospects。这一步也可直接用潜客详情的「AI 找联系人」触发邮箱查找 Webhook。

> 合规提醒：Webhook 侧对接的第三方数据源需自行确保数据来源合法、遵守其服务条款；系统只负责把返回的公开信息接进漏斗，不做任何抓取或反爬绕过。浏览器直连需目标地址允许跨域（CORS），或在自动化平台侧开启。
