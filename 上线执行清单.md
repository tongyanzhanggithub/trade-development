# 上线执行清单

把系统从「本地演示」推到「真实开发客户」的分步清单。含：① 四阶段勾选清单，② 发信域名 DNS（SPF/DKIM/DMARC）配置示例，③ n8n 六个 Webhook 的接口规格（字段与代码一致）。

> 关键路径（缺一不可才能真跑）：**AI Key → 发信域名+预热 → SMTP 发信 → IMAP+n8n 收回信**。其余都是加速器。
> 发送始终需人工审批；系统内置日限/预热/退订黑名单护栏，请勿关闭。

### Phase 2 · 发信基建（第 2–4 周，关键）

- [ ] 建「发信 Webhook」「拉取回复 Webhook」两条 n8n 工作流（见第三节）

- [ ] **按预热节奏起量**：前 1–2 周 ≤20 封/天，逐步加量（系统会提示）

## 二、发信域名 DNS 配置（SPF / DKIM / DMARC）

以专用发信域名 `mail.example.com`（或独立域名 `example-outreach.com`）为例，在域名 DNS 处添加：

**SPF**（TXT 记录，声明谁能替你发信）
```
类型: TXT
主机: @        （或你的发信子域）
值:   v=spf1 include:amazonses.com ~all
```
> 用其它 ESP 就把 `include:` 换成对应服务商的值；一条 SPF 记录里可含多个 include。

**DKIM**（由发信服务商生成，通常给你 3 条 CNAME）
```
类型: CNAME
主机: <token1>._domainkey
值:   <token1>.dkim.amazonses.com
（服务商会给 2–3 条，照抄即可）
```

**DMARC**（TXT 记录，告诉收件方如何处理未通过校验的信）
```
类型: TXT
主机: _dmarc
值:   v=DMARC1; p=none; rua=mailto:dmarc@example.com; pct=100; sp=none; aspf=r; adkim=r
```
> 起步用 `p=none`（只观察不拦截）；稳定 2–4 周、看 DMARC 报告无问题后，再逐步收紧到 `p=quarantine` → `p=reject`。

**自查**：用 `mxtoolbox.com` 或 `mail-tester.com` 发一封测试信，确认 SPF/DKIM/DMARC 三项全 pass，得分 ≥ 9/10 再起量。

## 三、n8n 六个 Webhook 接口规格

### 通用约定
- 系统对每个 Webhook 都发 **HTTP POST，`Content-Type: application/json`**。
- 请求体统一带信封字段，再合并各接口自己的字段：
  ```json
  { "type": "<接口名>", "sentAt": "2026-07-06T09:00:00.000Z", ...接口字段 }
  ```
- n8n 侧：用 **Webhook 触发节点**（HTTP Method = POST，Respond = "Using Respond to Webhook node"），末尾用 **Respond to Webhook** 节点返回下方约定的 JSON。
- 系统按 HTTP 状态码判断成败；**入库/发信类接口的实际数据取自返回的 JSON 字段**（下面每个接口标了「期望响应」）。
- 建议在 n8n 里先打印一次真实请求体，核对内层字段再写映射。

---

### 1) 搜索采集 `type: "search"`
系统 → 你：请求
```json
{ "type": "search", "sentAt": "...",
  "campaign": { "product": "...", "markets": "...", "customerType": "...", "valueProps": "..." },
  "searchPlan": [ { "channel": "...", "query": "...", "market": "..." } ] }
```
你 → 系统：**期望响应**（读取 `prospects` 数组）
```json
{ "prospects": [
  { "company": "ABC Trading", "website": "abc-trading.com", "market": "United States",
    "role": "Purchasing Manager", "email": "", "note": "疑似进口商" }
] }
```

### 2) 邮箱查找/验证 `type: "enrich"`
系统 → 你：请求
```json
{ "type": "enrich", "sentAt": "...",
  "company": "ABC Trading", "domain": "abc-trading.com",
  "market": "United States", "role_hint": "importer distributor" }
```
你 → 系统：**期望响应**（任一命中即可）
```json
{ "contact_name": "John Doe", "contact_role": "Purchasing Manager",
  "email": "john@abc-trading.com",
  "email_candidates": [ { "email": "john@abc-trading.com", "confidence": 95, "pattern": "verified" } ],
  "phone": "+1...", "company_profile": "...", "fit_note": "...", "fit_score": 82 }
```
> 对接 Hunter/Apollo 时，在 n8n 里把它们的返回映射成上面的字段名。

### 3) 发信 `type: "send"`（SMTP）
系统 → 你：请求
```json
{ "type": "send", "sentAt": "...",
  "emails": [
    { "id": "...", "prospectId": "...", "company": "ABC Trading",
      "email": "john@abc-trading.com", "subject": "...", "body": "...",
      "step": "首封", "dueDate": "2026-07-06" }
  ] }
```
n8n：遍历 `emails`，用 `email`（收件人）+ `subject` + `body` 经 SMTP 节点发送。
你 → 系统：返回 **HTTP 200** 即视为已派发（系统随后把这些邮件标「已发送」）。

### 4) WhatsApp `type: "whatsapp"`
系统 → 你：请求
```json
{ "type": "whatsapp", "sentAt": "...",
  "messages": [ { "id": "...", "prospectId": "...", "phone": "+1...", "message": "...", "company": "..." } ] }
```
n8n：遍历 `messages`，经 WhatsApp Business API 发送。返回 200 即视为已派发。

### 5) CRM 同步 `type: "crm"`
系统 → 你：请求
```json
{ "type": "crm", "sentAt": "...",
  "prospects": [ { "company": "...", "email": "...", "stage": "询盘", "amount": 0, "market": "..." } ] }
```
n8n：写入你的 CRM。返回 200 即可。

### 6) 拉取回复 `type: "inbound"`（真实客户回信入口，最关键）
系统 → 你：请求（`since` 为上次拉取时间，首次为 `null`）
```json
{ "type": "inbound", "sentAt": "...", "since": "2026-07-06T08:00:00.000Z" }
```
n8n：从 IMAP 收件箱拉取 `since` 之后的新回信。
你 → 系统：**期望响应**（读取 `replies` 数组）
```json
{ "replies": [
  { "from_email": "john@abc-trading.com", "from_name": "John Doe",
    "company": "ABC Trading", "market": "United States",
    "text": "客户回信正文……", "channel": "email",
    "at": "2026-07-06T08:30:00.000Z", "phone": "" }
] }
```
字段兼容：`from_email` 可用 `email`；`text` 可用 `body`。系统按邮箱/公司名匹配线索，匹配不上会**自动补建线索**绝不丢信；入库即联动「回复即停 / 退订黑名单 / 意图推进 / 风险扫描 / AI 应答」。收件箱「拉取回复」按钮与自动驾驶（每分钟）都走此接口。

- [ ] 首两周严格 ≤20 封/天，无退信激增
- [ ] 每封邮件含退订方式；退订黑名单开启
