# 本地部署说明

有两种用法：**① 桌面应用版（推荐，像真软件）** 或 **② 轻量版 `启动.bat`（浏览器里跑）**。两者是**各自独立的数据空间**，数据不互通——用其中一种就一直用它，需要搬数据走"导出/导入备份"。

---

## ① 桌面应用版（推荐）

双击一个图标弹出独立窗口，没有网址栏、没有黑色终端、不占浏览器。前提：电脑装有 **Node.js**（[nodejs.org](https://nodejs.org) 下载安装，你已装）。

**最省事：双击 `打包.bat`（一键打包）**

它会自动 `npm install` + `npm run dist`，完成后打开安装包所在的 `dist-desktop\` 文件夹。首次需联网、约几分钟。里面的 **`重庆外贸自动获客 Setup 1.0.0.exe`** 双击安装即可——会创建桌面快捷方式和开始菜单项，都带自定义图标（墨绿底金色纸飞机，已随包内置，无需额外步骤）。以后**双击桌面图标**打开，跟普通软件一样。

**或手动两条命令**（在 `trade-development` 目录）：

```
npm install          # 装依赖（首次需联网，几分钟）
npm run dist         # 打包出 Windows 安装包（在 dist-desktop\）
```

**只想立刻用、先不做安装包**：`npm install` 之后直接 `npm start`，立即以应用窗口形式打开（开发模式）。

**关于数据**：桌面版数据存在应用自己的空间里（菜单 `文件 → 打开数据存放位置` 可看到），不绑端口、不受清浏览器缓存影响。备份/搬迁仍走「设置 → 数据与备份 → 导出 / 导入」。

> 从浏览器版切到桌面版：先在浏览器版点「导出全部数据」，打开桌面版后用「导入备份」选那个 JSON 即可把线索/黑名单/配置全带过来。

---

## ② 轻量版：启动.bat（浏览器里跑）

双击 `trade-development` 目录里的 **`启动.bat`**：它会在固定端口 4174 起一个本地服务并自动打开浏览器。**用系统期间保持那个黑色命令行窗口开着**，关掉它 = 关掉系统。

前提：电脑装有 Python（Windows 商店装一个即可）。没有 Python 也可以用任何静态服务器指向本目录，但**端口必须固定**（见下）。

## 三条必须知道的规矩

1. **端口不能换（仅浏览器版）**。浏览器版所有数据（线索、队列、退订黑名单、API Key、配置）存在浏览器 localStorage 里，绑定"地址+端口"。今天用 4174、明天用 8000，数据不会丢但你会"看不到"。永远用 `启动.bat`，就永远是 4174。（**桌面应用版无此问题**——它不绑端口。）
2. **每周备份一次**。浏览器清缓存/重装/换浏览器/换电脑 = 数据没了。点右上角「导出全部数据」（或 设置 → 数据与备份 → 立即备份）保存 JSON 文件；系统超过 7 天没备份会开屏提醒，设置页能看到存储占用和上次备份时间。换电脑时把备份 JSON 的内容导入即可恢复。
3. **只开一个标签页**。开两个标签页同时操作会互相覆盖数据，系统检测到会弹红色横幅提醒。

## 自动化的边界

- 自动驾驶/周期补量是**浏览器里的定时器**：浏览器关了就停。日常习惯是"上班开着它，让它自己跑"；要 7×24 无人值守，需把采集/发送放到 n8n 等外部平台（配合各 Webhook + 外部 cron）。
- **发送永远等你人工审批**，这是设计而不是缺陷。

## 接真实服务（可选，按需逐个接）

在「设置」切到 Webhook 模式后逐个填入（每个都有"测试连接"）：

| 接口 | 作用 | 常见对接 |
| --- | --- | --- |
| 搜索采集 | 真实数据源找客户 | n8n + SerpAPI / 海关数据 |
| 邮箱查找/验证 | 真实联系人与验证邮箱 | Hunter / Apollo / ZeroBounce |
| 发信 | 真实发邮件 | SMTP / SendGrid / Mailgun |
| WhatsApp | 真实发送 | WhatsApp Business API / 360dialog |
| CRM 同步 | 客户库双向同步 | Twenty / EspoCRM / HubSpot |
| **拉取回复** | **真实客户回信进收件箱** | n8n IMAP 收件（见 README「拉取回复接口」） |

其中「拉取回复」最关键——不接它，客户回了信系统不知道，意图识别/AI 应答/热意向移交都空转。接好后：收件箱手动点「拉取回复」，或开自动驾驶（每分钟自动拉一次）。

## 安全提醒

- Anthropic API Key 只存本机浏览器、随请求直连 Anthropic，不经过任何第三方；但**不要把备份 JSON 发给别人**（里面含 Key）。
- 多人使用请部署服务端代理管理 Key，浏览器直连仅适合个人本机使用。
